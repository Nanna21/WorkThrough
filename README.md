# 워크쓰루 (Work Through)

> **"번거로운 화면 분할을 대신해주고, 4개의 빈 공간에 원하는 사이트를 한 번에 불러올 수 있는 작업 편의성 웹 서비스입니다."**

---

## 1. 기획 배경 및 해결 과제
* **현재 상황 및 문제점: 매일 반복적으로 해야하는 작업에 대한 불편함.**
* **우리가 제공할 해결책: 자동화 기능을 만들어서 매일 해야하는 반복적인 작업에 대한 번거로움을 줄인다.** 

## 2. 타겟 유저 페르소나
* **인구통계학적 특성: 현대인**
* **불편함: 작업 환경 구성에 대한 문제점**
* **기대하는 것: 작업의 효율화** 

## 3. 핵심 기능
> 프론트엔드 환경에서 3주 내에 구현할 수 있는 핵심적인 기능 딱 3가지를 정의합니다.
1. **[화면 분할]**: 작업창의 크기가 자유로운 4분할 화면하는 기능
2. **[실시간 화면 저장]**: 현재 작업중인 화면을 실시간으로 저장하여 수동 저장에 대한 불편함 감소시키는 기능
3. **[데이터 불러오기]**: 저장된 화면과 즐겨찾기해둔 사이트를 한 번에 불러오는 기능

## 4. 화면 설계 및 유저 플로우
* 사용자가 우리 웹사이트에 접속하여 목표를 달성하기까지의 클릭 흐름을 단계별로 작성합니다.
  1. `메인 페이지` 접속
  2. 원하는 카테고리 파일 찾기
  3. 파일 불러오기
  4. 즐겨찾기 파일 열기
  5. 사이트 불러오기
  6. 개인별 세팅


## 5. 기술 스택 및 아키텍처
